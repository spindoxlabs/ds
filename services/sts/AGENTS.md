# ds-sts — Agent Guide

## Service identity

- **Role**: Security Token Service — issues ES256 Self-Issued JWTs for DCP identity proofs
- **Language**: Python 3.12, FastAPI
- **Ports**: 38080 (provider), 38081 (consumer)
- **URLs**: `https://sts-provider.dataspaces.localhost`, `https://sts-consumer.dataspaces.localhost`
- **Database**: none

## Source layout

```
src/sts/
├── main.py        FastAPI app — /token, /jwks, /.well-known/openid-configuration, /health
├── config.py      Pydantic settings (StsSettings)
└── token.py       ES256 JWT generation logic
```

## Key files for common tasks

| Task | Files to touch |
|------|---------------|
| Change token claims or format | `token.py` |
| Add new endpoints | `main.py` |
| Change config/env vars | `config.py` |

## Coding conventions

- Each participant runs their own STS instance with a unique DID and private key
- Token format: ES256 JWT with `iss`, `sub` (both = participant DID), `aud`, `iat`, `exp`, `jti`, `bearerAccessScope`
- Private keys are EC P-256 JWK files mounted at `STS_PRIVATE_KEY_PATH`
- The `/token` endpoint implements OAuth2 `client_credentials` grant
- JWKS endpoint returns the public key derived from the private key at startup
- This service is stateless — no database, no persistent state

## Configuration

| Env var | Default | Purpose |
|---------|---------|---------|
| `STS_PARTICIPANT_DID` | — | `did:web:` URI for this participant |
| `STS_PRIVATE_KEY_PATH` | — | Path to EC P-256 JWK private key file |
| `STS_CLIENT_ID` | — | OAuth2 client ID (= participant DID) |
| `STS_CLIENT_SECRET` | `insecure-dev-secret` | OAuth2 client secret |

## Integration points

- **Called by**: EDC connectors during DSP negotiation (to obtain SI tokens)
- **No downstream dependencies**
- Key files generated by `scripts/gen-keys.sh`
